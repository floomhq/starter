# Deployment note for the starter site

This directory is its own Vercel project: `starter-pack-2026-05-10` (linked via `.vercel/project.json`).

`floom.dev`, `www.floom.dev`, and `starter.floom.dev` point at this project.

Vercel aliasing on 2026-05-11 points the public starter launch domains at this project:

- `floom.dev`, `www.floom.dev`, `starter.floom.dev` -> Vercel project `starter-pack-2026-05-10`
- `relay.floom.dev`, `skills.floom.dev` -> Vercel project `skills-minimal-deploy`

## To deploy

ALWAYS deploy from inside this directory:

```bash
cd /root/floom-starter/site
vercel deploy --prod --yes
```

Do NOT run `vercel deploy` from `/tmp/floom-wireframes/` or `/root/floom-starter/` — that can push a different project or the wrong root.

## Background

A previous chained agent created the separate `starter-pack-2026-05-10` Vercel project for the starter-pack static site. Subsequent edits to other directories do not auto-propagate. Always deploy from this folder when updating the public starter site.
