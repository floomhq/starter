# Deployment note for floom.dev

This directory is its own Vercel project: `starter-pack-2026-05-10` (linked via `.vercel/project.json`).

floom.dev points at THIS project — NOT the parent /tmp/floom-wireframes.

## To deploy

ALWAYS deploy from inside this directory:

```bash
cd /tmp/floom-wireframes/starter-pack-adam-merged-2026-05-11
vercel deploy --prod --yes
```

Do NOT run `vercel deploy` from `/tmp/floom-wireframes/` — that pushes to a different project and floom.dev will not pick up the change.

## Background

A previous chained agent created the separate `starter-pack-2026-05-10` Vercel project specifically to host floom.dev. Subsequent edits to other directories did not auto-propagate. Always deploy from this folder.
