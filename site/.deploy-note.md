# Deployment note for the starter site

This directory is its own Vercel project: `starter-pack-2026-05-10` (linked via `.vercel/project.json`).

`starter.floom.dev` points at this project.

`floom.dev` does not point at this project. Vercel domain inspection on 2026-05-11 showed:

- `floom.dev`, `www.floom.dev` -> Vercel project `floom-prod`
- `starter.floom.dev` -> Vercel project `starter-pack-2026-05-10`
- `relay.floom.dev`, `skills.floom.dev` -> Vercel project `skills-minimal-deploy`

## To deploy

ALWAYS deploy from inside this directory:

```bash
cd /root/floom-starter/site
vercel deploy --prod --yes
```

Do NOT run `vercel deploy` from `/tmp/floom-wireframes/` or `/root/floom-starter/` — that can push a different project or the wrong root.

## Background

A previous chained agent created the separate `starter-pack-2026-05-10` Vercel project for the starter-pack static site. Subsequent edits to other directories do not auto-propagate. Always deploy from this folder when updating `starter.floom.dev`.
